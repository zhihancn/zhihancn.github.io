操作系统类型

  1. 多道批处理系统
  2. 分时系统
  3. 实时系统
  4. 网络操作系统
  5. 分布式操作系统
  6. 多处理机操作系统
  7. 嵌入式操作系统

1,2,3,4,5为5种基本操作系统

网络操作系统

计算机网络的特征

  1. 分布性。
    • 网上节点机可位于不同地点，各自执行自己的任务。
  2. 自治性。
    • 每台计算机都有自己的内存、I/O设备和操作系统，能够独立完成自己承担的任务。
  3. 互连性。
    • 利用互连网络把不同地点的资源连接在一起，在统一的网络操作系统的控制下，实现网络通信和资源共享。
  4. 可见性。
    • 计算机网络中的资源对用户是可见的。用户任务通常在本地机运行，利用网络操作系统提供的服务共享其他主机上的资源。

计算机网络通常是客户机-服务器结构

  • 一台或几台功能强大的机器作为服务器，其他机器是单用户的工作站，即客户机。
  • 服务器上运行网络操作系统，客户机上运行本地操作系统。

网络操作系统建立在各个主机的本地操作系统基础之上，提供网络资源共享、网络通信、网络管理、网络服务和网络接口等功能。

  • 对整个网络实施管理。为用户提供统一方便的网络接口。
  • 对用户屏蔽了各个主机对相同资源的不同存取方法。
  • 网络用户只有通过它才能获得网络所提供的各种服务。

网络OS特征

  1. 接口一致性。
    • 例如，网用户可用同一命令存取本地文件或远程文件，设备可用一致的路径进行操作。
  2. 资源透明性。
    • 每用户不必知道每个操作需要哪些资源的支持。网络OS了解整个网络中共享资源的状态和使用情况，根据用户的要求自动做出最优选择。
  3. 操作可靠性。
    • 对全网的共享资源进行统一管理和调度，某处资源出现故障，可分配其他同类资源完成用户请求。
  4. 处理自治性。
    • 每台主机都有独立的处理能力。
  5. 执行并行性。
    • 实现本机上的进程并发，实现各节点机上进程的并行。例如，可通过远程命令在节点机上完成指定任务，在本地机上同时执行其他操作。

分布式操作系统

特征

  1. 透明性。
    • 对用户隐藏系统内部的细节，例如资源的位置、活动的迁移。
  2. 灵活性。
    • 可根据用户的需求和使用情况，方便地修改和扩充系统。
  3. 可靠性（包括可用性、安全性、容错性）。
    • 系统中某台机器出现故障，其他机器替代工作。
  4. 高性能。
    • 速度快、相应及时、资源利用率高、网络通信能力强。
  5. 可扩充性。
    • 根据使用环境和应用的需要，方便扩充或缩减规模。

缺点

  • 供分布式系统使用的软件少（操作系统、编程语言等）。
  • 通信网络会出现饱和，安全性问题。

多处理机操作系统

系统特点：多个CPU共享总线、时钟、内存、外部设备。

[!TIP]
产生并行条件

多处理机操作系统分为对称式和非对称式

  1. 对称式
    • 每个处理器运行同一个操作系统的副本，彼此通过共享内存实现通信。所有的处理器是对等的，没有主、从之分。任务负载较为平均，性能调节容易。
  2. 非对称式：主从模式
    • 主处理器：只有一个，运行OS，控制整个系统，管理整个系统的资源，为从处理器分配任务；
    • 从处理器：有多个，执行主处理器下达的指令或I/O处理。

嵌入式操作系统

优点：微小、实时、专业、可靠、易裁剪等。

在机器人、PDA、车载系统、家用电器、手机等通信设备上，都需要一个支持多道程序设计的环境，提供这种环境的操作系统称为嵌入式操作系统。

嵌入式操作系统的应用

操作系统的特性

  1. 并发(concurrency)
  2. 共享(sharing)
  3. 异步性(asynchronism)
  4. 虚拟(virtual)

并发

并发(concurrency)是两个或多个活动在同一时间间隔内发生。
并行(parallel)是指多个活动在同一时刻发生。

在单处理器情况下，处理多道程序时，宏观上并发，微观上交替执行。

程序的静态实体是可执行文件，而动态实体是进程（或称作任务），并发指的是进程。

共享

  1. 多个进程共享有限的计算机系统资源。
    • 多个进程同时占用内存，对内存共享。
    • 进程并发执行时，对CPU进行共享。
    • 进程提出对文件读写时，对磁盘进行共享。
  1. 为了保证进程能正常活动，系统必须对共享资源实施有效的管理

异步性

  1. 不确定性，指系统中各种事件发生顺序的不可预测性。
  2. 不确定性是系统中进程的并发性和资源共享所带来的必然结果。
  3. 进程间由于并发和共享，直接或间接地互相制约。
    • 进程“时走时停”，运行速度不可预知，比如分时系统。
    • 无论快慢，都不应该影响结果。

虚拟

  1. 虚拟指一个物理实体映射为若干个对应的逻辑实体。
  2. 虚拟是操作系统管理系统资源的重要手段，可提高资源利用率。

操作系统的结构

  1. 整体系统
  2. 层次系统
  3. 虚拟机系统
  4. 客户-服务器系统

整体系统:简单模块组合结构

  1. 这种组织结构是完全没有结构的，面向过程，无序的。
  2. 编写的操作系统是模块集合体，只要需要，每个过程都可随便调用任一其他过程。
  3. 模块之间的调用关系是一张大网，彼此直接联系。牵一发而动全身，可靠性低。

层次结构:面向大众一般使用的结构

按照功能和相互依存关系，把系统中的模块分为若干层，其中任一层模块都建立在它下面一层的基础上，任一模块只能调用比它低的层来得到服务，不能调用比它高的层。

内环权利高、贴近硬件

  1. 优点：
    • 结构清晰、提高系统的可靠性、可移植性、可维护性。
  2. 缺点：
    • 设计有困难
