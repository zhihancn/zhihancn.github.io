PAT乙级1005

薛某 2020-05-10 11:04:55

转自：https://zhuanlan.zhihu.com/p/62263029

卡拉兹(Callatz)猜想已经在1001中给出了描述。在这个题目里，情况稍微有些复杂。
当我们验证卡拉兹猜想的时候，为了避免重复计算，可以记录下递推过程中遇到的每一个数。例如对 n=3 进行验证的时候，我们需要计算 3、5、8、4、2、1，则当我们对 n=5、8、4、2 进行验证的时候，就可以直接判定卡拉兹猜想的真伪，而不需要重复计算，因为这 4 个数已经在验证3的时候遇到过了，我们称 5、8、4、2 是被 3“覆盖”的数。我们称一个数列中的某个数 n 为“关键数”，如果 n 不能被数列中的其他数字所覆盖。
现在给定一系列待验证的数字，我们只需要验证其中的几个关键数，就可以不必再重复验证余下的数字。你的任务就是找出这些关键数字，并按从大到小的顺序输出它们。
输入格式：
每个测试输入包含 1 个测试用例，第 1 行给出一个正整数 K (<100)，第 2 行给出 K 个互不相同的待验证的正整数 n (1<n≤100)的值，数字间用空格隔开。
输出格式：
每个测试用例的输出占一行，按从大到小的顺序输出关键数字。数字间用 1 个空格隔开，但一行中最后一个数字后没有空格。
输入样例：
6 3 5 6 7 8 11
输出样例：
7 6

题目分析：这道题我一开始用的就是普通的for循环去对比每一个被“覆盖”的数，相信大多数人用的是这个方法，也比较简单，但是我在网上看到大神写了一个我没有想到过的方法，感觉挺好的，就拿来分享一下，自己用这个代码的思维写了一下（不知道为什么gcc编译器下一直有段错误，clang下就是正确的，如果有大佬可以帮忙看一下的话求解答！），算法思想是建一个101大小的数组，数组里标记着这个数组是否是“关键数”，然后再一直做for循环，进行对比，改变标记状态。

#include <stdio.h>

int main(){
  int n=0;
  int a[101]={0};
  int k=0;
  scanf("%d",&k);
  for(int i=0;i<k;i++){
    scanf("%d",&n);
    a[n]=1;
  }
  for(int i=1;i<=100;i++){
    if(a[i]==1){
      for(int j=i;j>1;){
        if(j%2==0)
          j=j/2;
        else
          j=(j*3+1)/2;
        if(a[j]==1&&j<=100){
          a[j]=0;
          k--;
          if(j<i)
            break;
        }
      }
    }
  }
  for(int i=100;i>=1;i--){
    if(a[i]==1)
      printf("%d%c",i,--k?' ':'\0');
  }
  return 0;
}

printf(“%d%c”,i,–k?’ ‘:’\0’);这一句做输出超级好用啊！注意%c的空字符为’\0’，如果是%s的话就””可以了。
