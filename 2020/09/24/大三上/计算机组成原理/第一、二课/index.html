1.1 计算机系统简介

计算机体系结构 (例：有无乘法指令)

机器级程序员所见到的计算机系统的属性，即概念性的结构与功能特性

计算机组成(例：如何实现乘法指令)

实现计算机体系结构所体现的属性

计算机实现是计算机组成的物理实现

CPU 5个专用寄存器

  1. MAR 存储器地址寄存器
  2. MDR 存储器数据寄存器
  3. PC 程序寄存器 (例：C++指示等)
  4. IR 地址寄存器
  5. PSWR 状态标志寄存器 (例：处理程序溢出等)

在这些中只有PC(程序计数器)、PSWR(状态标志寄存器)是汇编语言程序员可见的

存储器(MDR)内为指令(程序)或数据，此时我们并不知道次MDR装着什么

只有到控制器(IR)查看并执行

1.2计算机的基本组成

一、冯·诺依曼计算机的特点

  1. 计算机由五大部件组成
  2. 指令和数据以同等地位存于存储器，可按地址寻访
  3. 指令和数据均用二进制表示
  4. 指令由操作码和地址码组成
  5. 存储程序（指令在存储器内按顺序存放）
  6. 以运算器为中心，输入设备与存储器间的数据传送通过运算器完成

冯·诺依曼计算机硬件框图

各个设备的主要功能：

设备名称 主要作用
输入设备 将信息转换成机器能识别的形式
储存器 存放数据和程序
运算器 算术运算逻辑运算
控制器 指挥程序运行
输出设备 将结果转换成人们熟悉的形式

3.2 计算机的解题过程

3.2.1 存储器的基本组成

  1. MAR (Address)

    • 存储器地址寄存器
    • 反映存储单元的个数
    • 存储单元个数= 2^(MAR)(因为每个存储元件都有2种情况)

  2. MDR(Data)

    • 存储器数据寄存器

    • 反映存储字长

    • 存储字长=MDR

1.3 计算机硬件的主要技术指标

  1. 机器字长

    • CPU 一次能处理数据的位数

    • 与 CPU 中的 寄存器位数 有关

字长越长，表数范围越大，精度越高。

字长会影响机器的运算速度。

字长对硬件（ALU、数据总线、存储字长的位数）的造价有较大的影响。

  1. 存储容量
    • 存放二进制信息的总位数

1B(yte)=8b(it)

1KB=2^10B

  1. 字地址：

    • 存放一个机器字的存储单元，称为字存储单元，相应的单元地址叫字地址。

  2. 字节地址：

    • 存放一个字节的存储单元，称为字节存储单元，相应的单元地址叫字节地址。

  3. 字寻址计算机：

    • 计算机中可编址的最小单位是字存储单元

  4. 字节寻址计算：

    • 计算机中可编址的最小单位是字节存储单元。

  5. 存储单元：

    • 存放一个机器字的所有存储元集合。

  6. 存储字长：

    • 存储单元中存放二进制代码的总位数
    • 内存中存储字的长度。
